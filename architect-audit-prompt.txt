## OWASP Top 10 Checklist

For each file, evaluate against ALL of the following:

1. **A01 — Broken Access Control**: Are there any access control issues? Is client-side data being exposed? Are server actions properly guarded?
2. **A02 — Cryptographic Failures**: Any hardcoded secrets, API keys, or sensitive data exposed in client-side code?
3. **A03 — Injection**: Any SQL, XSS, or command injection vectors? React auto-escapes by default but evaluate carefully.
4. **A04 — Insecure Design**: Any architectural or design-level security weaknesses?
5. **A05 — Security Misconfiguration**: Next.js config issues, missing security headers, debug mode enabled?
6. **A06 — Vulnerable and Outdated Components**: Are there known vulnerable packages? (Not applicable to the new code since we didn't add packages)
7. **A07 — Identification and Authentication Failures**: Any auth-related issues?
8. **A08 — Software and Data Integrity Failures**: Any CI/CD issues, unsigned updates, or insecure deserialization?
9. **A09 — Security Logging and Monitoring Failures**: Any missing logging or monitoring?
10. **A10 — Server-Side Request Forgery (SSRF)**: Any URL-fetching that could be exploited?

## Additional Checks (Performance + UX)

- **Performance**: Any N+1 queries, unoptimized images, or large bundle impacts?
- **UX/Accessibility**: Are images missing alt text? Is the CTA accessible? Are focus states proper?
- **Bundle size**: Any new large dependencies?

## Output Format

Return findings organized as:

**🔴 CRITICAL** (file:line — description)
**🟠 HIGH** (file:line — description)
**🟡 MEDIUM** (file:line — description)
**🟢 LOW** (file:line — description)

If no issues found in any category, explicitly state "0 Critical, 0 High, 0 Medium, 0 Low".

Then provide a one-line summary of the overall security posture of this change.
